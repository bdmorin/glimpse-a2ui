# `[lit@0.8] CheckBox: input { width: 100% } stretches native checkbox into a full-width bar`

## Symptom

`a2ui-checkbox` renders a stretched gray bar instead of a recognizable checkbox + label pair. The native `<input type="checkbox">` is forced to `display: block; width: 100%` by the component's shadow CSS, which produces an unstyled input element occupying the full row width. The check state still toggles correctly (so `value` binding works), and the label is technically in the DOM, but the visual is dominated by the stretched input — labels are pushed below or visually masked, making CheckBox unusable for any UI that expects "checkbox-with-label" affordance.

Confirmed against:

- Upstream source: `renderers/lit/src/0.8/ui/checkbox.ts` (HEAD as of 2026-05-09)
- Vendored bundle in `bdmorin/a2glimpse` at host hash `0ff4d09524c9`

The visual harness goldens at `test/__snapshots__/0ff4d09524c9/checkbox.png` show the same stretched bars.

## Reproduction

Minimal A2UI v0.8 surface against the Lit renderer:

```json
{"surfaceUpdate":{"surfaceId":"vr","components":[
  {"id":"root","component":{"Column":{"children":{"explicitList":["q","ck1","ck2"]}}}},
  {"id":"q","component":{"Text":{"text":{"literalString":"Acknowledge:"}}}},
  {"id":"ck1","component":{"CheckBox":{"value":{"path":"/agree"},"label":{"literalString":"I agree to the terms"}}}},
  {"id":"ck2","component":{"CheckBox":{"value":{"literalBoolean":true},"label":{"literalString":"Subscribe to updates"}}}}
]}}
{"dataModelUpdate":{"surfaceId":"vr","contents":[{"key":"agree","valueBoolean":false}]}}
{"beginRendering":{"surfaceId":"vr","root":"root"}}
```

Expected: two checkboxes each followed by their text label, comparable to a plain HTML form.

Observed: small native checkbox glyph followed by a full-row gray rectangular bar (the stretched `<input>`); label text appears below the bar (or, depending on host theme/`additionalStyles`, may be visually masked entirely).

Grep against the published bundle confirms the rule is shipped:

```bash
grep -A2 'a2ui-checkbox' renderers/lit/dist/a2ui-lit.iife.js | grep -A1 'input{'
# → input{display:block;width:100%}
```

## Root cause

`renderers/lit/src/0.8/ui/checkbox.ts:49-52`:

```ts
static styles = [
  structuralStyles,
  css`
    * { box-sizing: border-box; }

    :host { display: block; flex: var(--weight); min-height: 0; overflow: auto; }

    input {
      display: block;
      width: 100%;
    }
    // …
  `,
];
```

The `input` rule is unscoped — it matches the `<input type="checkbox">` rendered at line 89-102 of the same file. `width: 100%` on a native checkbox produces a full-row stretched control with default browser styling, not a 16×16 px checkbox glyph.

`#renderField` at lines 82-107 emits the structurally-correct `<section><input/><label></label></section>` shape, but the input's stretched width pushes the label off the visual baseline and, combined with shadow-DOM theme classes that fill in `.description` margin-bottom, leaves the label visually disconnected or hidden.

## Spec reference

A2UI spec defines `CheckBox` as a labeled boolean toggle (see `specification/v0.8/components.md` — `CheckBox` carries both `value: BooleanValue` and `label: StringValue`). The renderer's stretched-input visual breaches the implicit contract that a "labeled checkbox" renders as a recognizable checkbox followed by readable label text.

## Suggested fix

Drop the unscoped `input { width: 100%; }` rule, or scope it explicitly to the `<input>` types this component should stretch (none — checkboxes shouldn't):

```diff
   input {
-    display: block;
-    width: 100%;
+    /* leave native checkbox at user-agent default; label flows next to it. */
   }
```

If `display: block` was intended for layout flow, prefer wrapping in a flex container at the `<section>` level rather than block-stretching the input itself:

```diff
-  input {
-    display: block;
-    width: 100%;
-  }
+  section {
+    display: flex;
+    align-items: center;
+    gap: 8px;
+  }
```

Either approach restores the native 16×16 px checkbox glyph and lets the label sit beside it. No breaking spec change.
