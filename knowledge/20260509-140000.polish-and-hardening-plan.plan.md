# a2glimpse Polish + Framework Hardening Plan

## Goals

1. Polish the visible surface (window chrome + host theme) so a2glimpse looks like a deliberate appliance, not a smoke test.
2. Harden the A2UI implementation framework per POC retro recommendation #1 (renderer load, test-gating, ready-signal).
3. Build visual-regression infrastructure so all subsequent work has a safety net.

## Non-Goals (this plan)

- v0.9 pivot — pinned to v0.8.
- MCP server wrapper — separate iteration.
- Agent skill — depends on a stable supported subset.
- `.app` packaging / signing / notarization.
- Cross-platform support — Mac-only, this host's environment specifically.

## Component Subset Stance

Goal: whole v0.8 component set. **Pragma:** if a single component takes more than ~1 session to fix cleanly, bail and log it in `DEV_LOG.md` under "deferred polish" with the reason, then move on. Vendored renderer means some classes of fix (markdown rendering, icon font bundling) require building host capability around the renderer — sometimes cheap, sometimes not.

### Phase 1 — Visual-regression harness (sequential, single agent)

**Why first now:** without it, hardening agents can regress visuals invisibly.

Deliverables:

- `test/fixtures/<name>.jsonl` — canonical A2UI v0.8 surface scripts. Start with: `button-only`, `card-text`, `modal`, `multiple-choice`, `text-field-form`, `tabs`.
- `test/visual.mjs` — for each fixture: spawn `a2glimpse` in test mode, dispatch the fixture, capture via mcporter `snap-happy.TakeScreenshot windowId:<id>`, write `test/__snapshots__/<fixture>.png`.
- Two modes: `--update` (write goldens) and default (compare). Pixel-diff via ImageMagick `compare` at 0.1% threshold. Host-page wrapper carries `--test-mode` CSS to neutralize caret/focus/animation non-determinism.
- A `--test-mode` flag added to `a2glimpse.swift` that locks geometry (fixed window size, no follow-cursor), so captures are deterministic on this host.
- Bake the vendored renderer's content hash into the snapshot directory name (e.g. `__snapshots__/<short-hash>/<fixture>.png`) so a renderer bump is a deliberate re-baseline, not silent drift.

Acceptance: `node test/visual.mjs --update` writes all goldens, then `node test/visual.mjs` (no flag) passes pixel-diff (≤0.1%) against them across three back-to-back runs.

### Phase 2 — Framework hardening (parallel worktrees)

Three independent slices. Each agent works in its own worktree.

| Worktree | Task | Touches |
|---|---|---|
| `wt/harden-renderer-load` | Drop cwd renderer-host fallback. Resolve from bundled path adjacent to the binary; fail loudly if missing. | `src/a2glimpse.swift` |
| `wt/harden-test-gating` | Synthetic-click and any other test-only paths require `--test-mode` (or env). Default command dispatch cannot reach them. | `src/a2glimpse.swift` |
| `wt/harden-ready-signal` | Add a renderer-host `ready` postMessage. Swift waits for it before emitting stdout `ready`. Update `a2glimpse.mjs` consumer. | `src/a2glimpse-host.html`, `src/a2glimpse.swift`, `src/a2glimpse.mjs` |

Each agent must:

1. Verify smoke (`npm test`) passes pre- and post-change.
2. Verify visual-regression baselines unchanged (or re-bless deliberately, with rationale in commit body).
3. Append a `DEV_LOG.md` entry.
4. Append an `AUDIT_LOG.md` entry with `Worktree: wt/<slice>` and `Outcome:`.
5. If blocked, write a retro in `knowledge/<timestamp>.<slice>.retrospective.md` and reference it from AUDIT_LOG.

### Phase 3 — Component polish (parallel worktrees)

After visual-regression baselines exist. One worktree per component cluster:

- `wt/polish-button-card-text`
- `wt/polish-modal`
- `wt/polish-icon` (likely DEFERRED — Material Symbols bundling)
- `wt/polish-multiplechoice-checkbox`
- `wt/polish-textfield-slider`
- `wt/polish-tabs`
- `wt/polish-usagehint-markdown` (likely DEFERRED — markdown engine)

Per-component agent loop:

1. Ensure a fixture for the component exists; render and snap.
2. Identify visual issue.
3. Fix in `a2glimpse-host.html` theme/host-shim only — never edit vendored Lit source.
4. Re-snap; commit golden update with rationale.
5. **Bail rule:** >1 session of fighting → stop, append DEV_LOG "deferred polish" entry, mark AUDIT_LOG `Outcome: DEFERRED`, move on.

### Phase 4 — Window chrome (Swift, sequential)

After host theme settles. Title bar styling, default size, content padding, dark-mode awareness. Reuses existing `resize`/`title`/`show` lifecycle commands; no new commands.

### Phase 5 — Merge & integrate (orchestrator only)

Merges are owned by the main session, not by worktree agents. Order: Phase 2 (any order, all three) → Phase 3 clusters (resolve any CSS conflicts in `a2glimpse-host.html`) → Phase 4. Re-run `npm test` and visual-regression after each merge.

## Conventions for Worktree Agents

- **Worktree naming:** `wt/<slice>` matching the table above. Branch name == worktree name.
- **Commit cadence:** small, themed commits per fix. Don't squash inside a worktree.
- **Trust boundary is load-bearing.** Never reintroduce `html`, `file`, or `eval` on the public stdin surface, even temporarily.
- **Vendored renderer is read-only.** All visual fixes happen in the host page wrapper / theme CSS.
- **Tooling:** mcporter for snap-happy (`mcporter call snap-happy.TakeScreenshot windowId:<id> --output json`). Don't try to add a native MCP server.
- **Retro on block:** standard timestamped retro filename, link from AUDIT_LOG entry.
- **Defer in DEV_LOG:** anything skipped under the bail rule goes in DEV_LOG with reason + estimated effort to revisit.
- **Timeouts are HARD_BLOCK, never OK.** Do not paper over a dispatched-agent timeout with a silent retry or a "different approach." Capture timeout duration, last known activity, partial output, and dispatch context. Surface it to the orchestrator/Brian. Investigate root cause (wedged subprocess, deadlocked tool, prompt size, subagent type mismatch) before any retry. AUDIT_LOG: `Outcome: HARD_BLOCK` with retro link.

## Risks & Mitigations (recorded)

- **Renderer bundle drift:** mitigated by content-hash in snapshot directory name (Phase 1).
- **WKWebView visual variation across environments:** plan is host-targeted to this machine; cross-host parity is not a goal. Documented in Phase 1.
- **Parallel agent merge conflicts in `a2glimpse-host.html`:** orchestrator owns merges; agents stay in their lane. Risk accepted.
- **HARD/SOFT_BLOCK/FAIL:** captured via standard retro + AUDIT_LOG lineage.
