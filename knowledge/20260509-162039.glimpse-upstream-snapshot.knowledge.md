# Glimpse Upstream Snapshot

This is the **baseline before any fork edits**. The repo at `/Users/brahn/src/github.com/bdmorin/glimpse-a2ui` is verbatim upstream HazAT/glimpse as of clone date — no glimpse-a2ui modifications have been made yet.

## What Glimpse Is

Native WebView window controlled via JSON Lines stdio. Three platform binaries, one Node wrapper, one shared protocol. License: **MIT**. Latest upstream release: **v0.8.0** (2026-04-19). Maintainer: **HazAT (solo)**, 99 commits, 708 stars.

## Architecture

| Platform | Backend | Language | Source |
|----------|---------|----------|--------|
| macOS | WKWebView | Swift | `src/glimpse.swift` (~420 lines) |
| Linux | GTK4 + WebKitGTK | Rust | `src/linux/` |
| Linux (alt) | Chromium CDP | JS | `src/chromium-backend.mjs` |
| Windows | WebView2 + .NET 8 | C# | `native/windows/` |

Node wrapper (`src/glimpse.mjs`) is convenience only — any process that can spawn + pipe JSON can drive the binary directly.

## Wire Protocol (Upstream)

### Stdin → binary (commands)

```jsonc
{"type":"html",  "html":"<base64-encoded HTML>"}
{"type":"eval",  "js":"..."}
{"type":"file",  "path":"/abs/path.html"}
{"type":"get-info"}
{"type":"show",  "title":"..."}
{"type":"title", "title":"🔴"}
{"type":"resize","width":400,"height":300}
{"type":"follow-cursor","enabled":true,"anchor":"top-right","mode":"spring"}
{"type":"close"}
```

### Stdout → host (events)

```jsonc
{"type":"ready",   "screen":{...}, "screens":[...], "appearance":{...}, "cursor":{...}}
{"type":"info",    ...}
{"type":"message", "data":{...}}        // from window.glimpse.send()
{"type":"click"}
{"type":"closed"}
```

stderr carries `[glimpse] ...` debug lines and is not part of the protocol.

### Page-side bridge

Injected at document start as `window.glimpse`:
- `window.glimpse.send(data)` — POST JSON to host (main frame only)
- `window.glimpse.close()` — close window
- `window.glimpse.cursorTip` — for follow-cursor mode

## Lifecycle

1. Spawn process → WebView inits in background
2. First `ready` (blank page) — Node wrapper sends queued HTML
3. Second `ready` (user content loaded) — surfaced to caller
4. Host sends commands; binary emits async events
5. `close` (user or host) → `closed` event → process exits

## Conventions Inherited From Upstream (per AGENTS.md)

- **Single-file Swift.** All Mac code lives in `glimpse.swift`. Splitting is anti-pattern.
- **Zero deps.** Node wrapper uses only `node:` built-ins. Swift uses only Cocoa/WebKit.
- **ESM only** in Node-land.
- **Protocol-first.** New features land as JSONL commands/events; wrapper is sugar.
- **Compile on install.** `postinstall` runs `swiftc`. Forking the Swift source is a supported workflow.

## Platform Constraint Map (Honest)

| Feature | macOS | Linux (GTK) | Linux (Chromium) | Windows |
|---------|-------|-------------|------------------|---------|
| Basic HTML/eval | ✅ | ✅ | ✅ | ✅ |
| Status item / menu bar | ✅ | ❌ | ✅ (via CDP) | ❌ |
| Follow cursor | ✅ | Hyprland/X11 only | partial | ✅ |
| Spring physics | ✅ | Hyprland only | ❌ | ✅ |
| Open external links | ✅ | ❌ | ✅ | ❌ |

**Implication for fork:** Mac-first is the right path. The cross-platform parity story is already lumpy upstream — we don't owe it to anyone to maintain three native codebases on day one.

## Why This Snapshot Exists
