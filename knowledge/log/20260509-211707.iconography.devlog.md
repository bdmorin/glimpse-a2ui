# Iconography Cheatsheet — devlog

**Author:** Claude Opus 4.7 (worktree agent a3ded460ceddc815f)

**Context.** The bundled `MaterialSymbolsOutlined.woff2` carries the full ~3000-glyph Outlined catalog, but agents driving the canvas had no curated way to pick a name without web-fetching the Google Fonts catalog mid-task. Goal: ship a flat, scannable, intent-organized cheatsheet plus a discoverability hook in the agent SKILL.md.

**Did.**

- Verified the renderer's Icon shape from the vendored IIFE (`src/a2glimpse-host.html`, lines 5255-5257, 9104-9119): `IconSchema = { name: StringValueSchema }`. So the wire form is `{"Icon":{"name":{"literalString":"<snake_case>"}}}` — `name` is an A2UI StringValue, not a bare string. The renderer also auto-converts CamelCase to snake_case at render time, but we recommend snake_case at the source for readability of the JSONL log.
- Created `skills/a2glimpse/MATERIAL-ICONS.md` with ~280 ligature names across 10 categories aligned to agent intent: Status/feedback, Code/dev, File/IO, Navigation/arrows, Data/objects, Communication, Media/playback, Editing, People/accounts, Misc utility. Format: dot-separated dense category lines for scan-ability. Frontmatter per repo conventions (`type: reference`, full tag set).
- Added an `Iconography` section to `skills/a2glimpse/SKILL.md` between the wire-shape reference and the Cross-platform note. ~12 lines: explains the Icon shape, gives a one-line example, points at MATERIAL-ICONS.md, calls out the literal-fallback debugging tip.
- Did NOT touch `src/a2glimpse-host.html` (constraint — styling track owns it this session).

**Considered, rejected.**

- Alphabetical listing — rejected. The point is "what icon for what intent." Alphabetical is for catalog browsing, which is what fonts.google.com/icons is for.
- Exhaustive enumeration of the 3000-glyph catalog — explicitly out of scope per the bail rule. Curated > exhaustive.
- Validating each ligature against the woff2 — `font-display: block` falls back to ligature text on miss, so misses are visually obvious in dev. Picked from the well-established Material Symbols Outlined set; avoided niche/decorative names.
- Putting the cheatsheet inside SKILL.md — would balloon the skill body. Sibling reference file keeps SKILL.md focused on patterns, with a one-liner pointer.

**Open / next.**

- The styling track may want to surface a host-level icon-color CSS variable; that's separate. The cheatsheet is renderer-agnostic.
- Future: if the renderer ever exposes weight/fill axes, this doc would need an axis section. The bundled font is single-weight 400 outlined only; not a near-term concern.
- Could add a Material Symbols → emoji parity table for non-icon-capable surfaces. Defer until requested.

**Trust boundary.** Untouched. No html/file/eval, no public stdin commands, vendored IIFE read-only.
