# Phase 3 — Modal polish (retry)

## Author
Claude Opus 4.7 (orchestrator-dispatched sub-agent), retry of an earlier slice
that DEFERRED on shadow-DOM scoping.

## Context
First Modal slice DEFERRED because the agent only considered wrapper CSS as a
lever and concluded the renderer's shadow-DOM hardcodes were unreachable. The
new vendored-renderer-pathologies knowledge enumerates three levers
(CSS custom properties, `additionalStyles.<Component>`, prototype patching) and
also calls out that the Modal fixture only captures the closed entry trigger —
the dialog never mounts, so even a perfect polish would be invisible to the
visual harness.

## Did
Two concerns, two fixes, both wrapper-side:

### 1. additionalStyles.Modal (Lever 2) — modal-panel chrome

The vendored Modal renders as `<dialog><section class=...Modal.element style=${additionalStyles.Modal}>...</section></dialog>`.
The `<dialog>` itself is locked by shadow CSS (`dialog { padding:0; border:none; background:none }`) with no var() indirection — Lever 1 (custom props) cannot reach it. But the inner `<section>` is the documented theme hook; Lever 2 reaches it via the inline `style` attribute the renderer already emits.

Promoted the inner section to the visible "modal panel": white surface (using
`var(--md-sys-color-surface)` already defined for other components), 1px
outline-variant border, 12px radius, level-2 shadow, 20×22 padding,
min-width 280, max-width min(90vw, 480px). The dialog becomes a transparent
positioning container; the section is the card.

Applied in `defaultTheme.additionalStyles.Modal` (post-IIFE config object that
the IIFE itself reads — not an IIFE edit).

### 2. Test-mode auto-open (fixture-side, no protocol change)

The fixture problem: a closed-state Modal renders only `<section><slot name="entry"></slot></section>` until clicked. The `<dialog>` doesn't mount until `_showModal` flips. The visual harness needs the open state.

Picked Approach 2 (auto-open via wrapper script) over Approach 1 (sibling fixture). Reasons:
- A2UI v0.8 has no spec message for "open this modal". An open-state fixture would require either a synthetic stdin command (punches the trust boundary the fork exists to close) or a new public message type (scope expansion). Neither is acceptable.
- Auto-open as a wrapper-side prototype patch, gated on `body[data-test-mode]`, keeps production identical — the user clicks the trigger themselves. Test-mode flag is set by Swift's `--test-mode` path only; not reachable from stdin.

Implementation: `customElements.whenDefined('a2ui-modal').then(...)` wraps `prototype.updated`. On the first update where `body[data-test-mode]` is set, find the closed-state `<section>` in the shadow root and `.click()` it. The click handler is the renderer's own `_showModal = true` setter; the next render path renders the `<dialog>` and `requestAnimationFrame` calls `el.showModal()` per the IIFE. One-shot via instance flag.

## Considered + rejected
- **Sibling `modal-open.jsonl` fixture.** A2UI v0.8 has no open-modal message;
  introducing one would be scope expansion + risk to the trust boundary.
  Auto-open via test-mode wrapper script is cleaner and doesn't touch the
  protocol.
- **Patching `Modal.render()` to fix the close button.** The hardcoded
  `<span class="g-icon">close</span>` (no Material Symbols font bundled) and
  the 20×20 native button are inside the renderer's emitted markup. Replacing
  them via Lever 3 is technically possible but is a rewrite of render output,
  not a polish patch. The Material Symbols icon-font issue is the same
  cross-cutting problem documented in earlier Phase 3 logs (icon slice
  DEFERRED on font bundling). Out of scope for this slice; proper fix is
  bundling the icon font or filing an upstream bug for `::part(close-button)`
  exposure.
- **Backdrop styling via `dialog::backdrop`.** Wrapper-side selector cannot
  reach into `a2ui-modal`'s shadow DOM where the dialog lives. The renderer
  doesn't define a `::backdrop` rule that we could override via custom prop,
  and the default browser backdrop is fine for now (semi-transparent gray).
