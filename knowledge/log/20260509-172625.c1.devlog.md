## What shipped

- `scripts/build-app.mjs` — assembles `dist/a2glimpse.app` from the existing
  `src/a2glimpse` binary + `src/a2glimpse-host.html`, generates `Info.plist`
  with `com.bdmorin.a2glimpse`, `LSUIElement: true`, `NSHighResolutionCapable`,
  `LSMinimumSystemVersion 11.0`, and version sourced from `package.json`.
  Best-effort generates a placeholder `AppIcon.icns` (flat slate-blue square
  with "a2g" centered) via inline `swift` + `sips` + `iconutil`. If any of
  those tools fail the script logs and continues without an icon — cosmetic,
  shouldn't block packaging.
- `npm run build:app` script entry.
- `src/a2glimpse.swift` — two changes:
  1. `loadRendererHost` now resolves dual-mode: `Bundle.main.url(forResource:)`
     first (finds `Contents/Resources/a2glimpse-host.html` when running
     inside a bundle), falling back to executable-adjacent for dev mode
     (`src/a2glimpse-host.html`). Comment in code documents WHY (one of the
     few WHY-comment cases per repo convention).
  2. Finder-launch detection at entry point. `wasLaunchedFromFinder()` tests
     `isatty(0) == 0` AND stdin is not a FIFO/socket (i.e. char device —
     /dev/null from launchd). Triggers an `NSAlert` explaining a2glimpse is
     an MCP appliance, then `exit(0)`. Pipe spawning (Node + `stdio:'pipe'`,
     shell pipelines) and TTY (interactive shell) flow through the normal
     stdin-reader path.
- `README.md` — new "Packaging (`.app` bundle)" section with build command,
  layout diagram, link to the onboarding tutorial, note that the bundle is
  unsigned and the bridge spawns `Contents/MacOS/a2glimpse` directly.
- `.gitignore` — added `dist/`.
- `knowledge/20260509-172625.apple-developer-onboarding.knowledge.md` —
  from-zero walkthrough: free vs paid dev account, Individual vs Org
  enrollment (recommends Individual unless LLC name needed on cert), cert
  hierarchy table, CSR generation via Keychain Access, certificate install +
  verify with `security find-identity`, fnox storage of TEAM_ID and
  app-specific password, full `codesign --options runtime --timestamp --deep`
  invocation, notarytool submission with `--wait`, stapling, `spctl`
  verification, common gotchas, and "when you don't need any of this" exit
  ramp for local-only use.
- `knowledge/INDEX.md` — new "Distribution / Packaging" section pointing at
  the onboarding doc.

## Verification

- `npm run build:macos` — succeeded (binary 245688 bytes).
- `npm run build:app` — succeeded; produced `dist/a2glimpse.app` with
  binary in `Contents/MacOS/`, host HTML + woff2 + AppIcon.icns in
  `Contents/Resources/`, Info.plist verified with `plutil -p`.
- `npm test` — passed (smoke test against `src/a2glimpse`). Finder-launch
  detection does not regress the Node-spawn path because Node attaches a
  pipe stdin (FIFO), which the detection explicitly excludes.
- `A2GLIMPSE_BINARY_PATH=...Contents/MacOS/a2glimpse npm test` — passed.
  Confirms bundle-aware path resolution works: the binary resolves
  `a2glimpse-host.html` from `Contents/Resources/` via `Bundle.main`.
- `open -W dist/a2glimpse.app` — alert appeared and held the foreground
  (timed out under `timeout 6` because the alert was waiting for OK,
  which is exactly the desired behavior).
